## Overview

Raffi supports conditional visibility for launcher entries. Use conditions to:

- Show entries only in specific desktop environments
- Display entries only when certain binaries are installed
- Conditionally show entries based on environment variables

You can specify multiple conditions per entry; all specified conditions must be met (logical AND) for the entry to be shown.

## Condition Types

### ifenvset

Shows the entry only if an environment variable is set (to any value).

**`ifenvset`** `string`

Environment variable name. Entry is shown only if this variable is set.


#### Example: Wayland-Only Entry

```yaml
rbw-pass:
  binary: rofi-rbw
  args:
    [
      "--selector wofi --clipboarder wl-copy --action copy-password --typer xdotool",
    ]
  description: "Password Manager"
  icon: password-manager
  ifenvset: WAYLAND_DISPLAY
```

This entry appears only when `WAYLAND_DISPLAY` is set (i.e., in Wayland sessions).

#### Example: Sway-Specific Entry

```yaml
cliphistory:
  binary: clipman
  args: ["pick", "-t", "wofi"]
  description: "Clipboard history"
  icon: accessories-clipboard
  ifenvset: SWAYSOCK
```

### ifenvnotset

Shows the entry only if an environment variable is **not** set.

**`ifenvnotset`** `string`

Environment variable name. Entry is shown only if this variable is not set.


#### Example

```yaml
x11-only:
  binary: xdotool
  description: "X11 Tool"
  ifenvnotset: WAYLAND_DISPLAY
```

### ifenveq

Shows the entry only if an environment variable equals a specific value.

**`ifenveq`** `array`

Array with exactly two elements: `[VARIABLE_NAME, EXPECTED_VALUE]`.
  
  Entry is shown only if `$VARIABLE_NAME == EXPECTED_VALUE`.


#### Example: GNOME-Only Entry

```yaml
keyboard-setting:
  binary: gnome-control-center
  args: [keyboard]
  description: "Keyboard setting"
  icon: preferences-desktop-keyboard-shortcuts
  ifenveq: [DESKTOP_SESSION, GNOME]
```

#### Example: X11-Only Entry

```yaml
snap-window-around:
  binary: wmctrl-resize-and-center
  args: [rotate]
  description: "Snap window around"
  icon: wmaround
  ifenveq: [XDG_SESSION_TYPE, x11]
```

#### Example: Desktop Environment Check

```yaml
gnome-shutdown:
  binary: gnome-session-quit
  args: [--power-off]
  description: "Shutdown / Poweroff"
  icon: gnome-shutdown
  ifenveq: [DESKTOP_SESSION, GNOME]

reboot:
  binary: gnome-session-quit
  args: [--reboot]
  description: "Reboot"
  icon: gnome-session-reboot
  ifenveq: [DESKTOP_SESSION, GNOME]
```

### ifexist

Shows the entry only if a specific binary exists in PATH.

**`ifexist`** `string`

Binary name to search in `$PATH`. Entry is shown only if the binary is found in PATH.

<Aside type="note">
`ifexist` performs a PATH lookup only. Absolute paths and shell-expanded paths (e.g., `~/bin/myapp`) are not supported and will be treated as missing.
</Aside>


#### Example: Binary Existence

```yaml
wally:
  binary: jumpapp
  args: ["-X", "wally"]
  description: "Wally"
  icon: wally
  ifexist: wally
```

Shown only if `wally` is installed in PATH.

#### Example: Multiple Entries for Same App

```yaml
evolution-mail:
  binary: jumpapp
  args: ["-t", "Evolution", "-X", "evolution"]
  description: "Evolution Mail"
  icon: evolution-mail
  ifexist: evolution

signal:
  binary: jumpapp
  args: ["-X", "signal", "signal-desktop"]
  description: "Signal"
  icon: signal-desktop
  ifexist: signal-desktop
```

## Combining with Other Fields

Conditions work with all other entry fields:

### Script with Condition

```yaml
my-script:
  script: |
    echo "Running on Wayland"
    env | grep WAYLAND
  description: "Wayland Info"
  icon: "info"
  ifenvset: WAYLAND_DISPLAY
```

### Disabled Entry with Condition

```yaml
optional-tool:
  binary: special-tool
  description: "Special Tool"
  ifexist: special-tool
  disabled: true
```

<Aside type="note">
Even if the condition passes, a `disabled: true` entry will be hidden.
</Aside>

## Validation Logic

Raffi evaluates conditions on startup. All specified conditions must pass (logical AND) for the entry to be shown:

1. **ifenveq**: Checks if variable exists and equals expected value
2. **ifenvset**: Checks if variable exists (any value)
3. **ifenvnotset**: Checks if variable does not exist
4. **ifexist**: Checks if binary is in PATH

If any condition fails, the entry is hidden.

### Automatic Binary Check

Even without conditions, Raffi automatically hides entries if:

- `binary` doesn't exist in PATH (for non-script entries)
- Script interpreter doesn't exist in PATH (for script entries)

```yaml
# Automatically hidden if firefox is not installed
firefox:
  binary: firefox
  description: "Firefox"
  icon: firefox
```

## Combining Multiple Conditions

Multiple condition fields can be combined on a single entry. All conditions are evaluated together using logical AND — the entry is shown only if every condition passes:

```yaml
# Shown only on GNOME + Wayland
gnome-wayland-tool:
  binary: my-tool
  description: "Tool (GNOME on Wayland)"
  ifenveq: [DESKTOP_SESSION, GNOME]
  ifenvset: WAYLAND_DISPLAY
```

If you need OR logic (show if any one of multiple conditions holds), create separate entries:

```yaml
# Entry for GNOME
gnome-tool:
  binary: my-tool
  description: "Tool (GNOME)"
  ifenveq: [DESKTOP_SESSION, GNOME]

# Entry for KDE
kde-tool:
  binary: my-tool
  description: "Tool (KDE)"
  ifenveq: [DESKTOP_SESSION, KDE]
```
